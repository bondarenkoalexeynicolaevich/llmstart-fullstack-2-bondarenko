# Задача 01: каркас FastAPI (фабрика, lifespan, health, `/v1`)

## Цель

Создать каталог `backend/` с минимально рабочим ASGI-приложением: **фабрика** `create_app()`, контекстный менеджер **`lifespan`** (пока без БД или с заглушкой `yield`, если БД подключается в задаче 02 — см. порядок ниже), **`GET /health`** с телом `{"status": "ok"}`, подключение **`APIRouter`** с префиксом **`/v1`** и тегами для будущих маршрутов (операции контракта **не** добавлять).

## Порядок относительно задачи 02

Допустимы два варианта исполнения:

1. **Сначала 01, затем 02:** в задаче 01 `lifespan` только `yield`; задача 02 переносит создание engine/session в `lifespan`.
2. **01+02 в одном PR агентом:** если удобнее, объединить создание engine в `lifespan` сразу в ветке после 02 — тогда в 01 оставить явный TODO в `lifespan` на подключение БД.

В планировании зафиксировано: **после обеих задач** при старте должна быть проверка БД.

## Файлы

- `backend/main.py` — `create_app()`, `app = create_app()` или документированная точка для uvicorn
- `backend/api/__init__.py`
- `backend/api/router.py` — `router = APIRouter(prefix="/v1", ...)`, `include_router` из `main`

## Критерии готовности

- Импорт приложения не вызывает side-effect кроме ожидаемого (создание app).
- `GET /health` → 200, JSON со статусом ok.
- Префикс `/v1` зарегистрирован; OpenAPI показывает пустой или почти пустой раздел v1 (без фиктивных POST под контракт, если не нужны для проверки).

## Не входит

- Pydantic-схемы контракта, авторизация `Bearer`, реальные POST.

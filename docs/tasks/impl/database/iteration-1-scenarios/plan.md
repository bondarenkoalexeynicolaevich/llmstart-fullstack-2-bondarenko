# Итерация 1: Пользовательские сценарии и требования к данным

**Область:** database  
**Статус:** завершена  
**Связь:** [tasklist-database.md](../../../tasklist-database.md)

## Цель

Зафиксировать без «технической кухни», что должны видеть и уметь студент и преподаватель в базовых сценариях, чтобы заложить основу под frontend и однозначно понять, какие сущности и связи нужны в данных.

## Ценность

Схема и API в следующих итерациях не проектируются «в вакууме» — каждая сущность привязана к сценарию.

## Definition of Done (итерация)

- Описаны 2–3 сценария для студента и 2–3 для преподавателя без деталей реализации.
- Каждый сценарий отображён на сущности из `docs/data-model.md` или зафиксированы пробелы (пробелов нет).
- `docs/data-model.md` содержит аннотации со ссылками на `docs/spec/user-scenarios.md`.

# Архитектурные решения (ADR)

ADR (Architecture Decision Record) — короткий документ, фиксирующий архитектурное решение: что выбрано, какие были альтернативы и почему сделан именно такой выбор.

## Зачем

- Сохраняет контекст решения для команды и будущего «я».
- Помогает не возвращаться к уже принятым вопросам.
- Делает архитектурный след проекта явным и читаемым.

## Формат каждого ADR

| Секция | Содержание |
|---|---|
| **Контекст** | Почему вообще встал этот вопрос |
| **Рассмотренные варианты** | Альтернативы с плюсами и минусами |
| **Решение** | Что выбрано и почему |
| **Последствия** | Что это решение влечёт за собой |

## Статусы

- `Предложено` — обсуждается, не принято
- `Принято` — действующее решение
- `Заменено` — вытеснено другим ADR (указать каким)

## Правила

- Один ADR — одно решение.
- После принятия ADR не редактируется. Если решение меняется — создаётся новый ADR со ссылкой на предыдущий.
- Нумерация сквозная: `adr-001`, `adr-002`, …
